[html] 页面导入样式时，使用link和@import有什么区别？

  1. link是html标签，@import是css提供的
  2. link引入的样式在页面加载时同时加载，@import则需要等页面加载完后再加载
  3. link没有兼容性问题，@import不兼容ie5以下
  4. link支持通过js操作DOM改变css样式，@import不可以。

参考：https://juejin.cn/post/6973562604581027853

圣杯布局：

middle的宽度为100%，left和right的宽度为100px。

三栏添加向左浮动，让它们显示在同一行。

会出现的问题：高度塌陷。

  • 解决方案1:在三栏布局设置overflow:hidden，触发BFC。
  • 解决方案2:底部footer区域设置清除浮动clear:both，解决高度塌陷问题。

解决完高度塌陷问题，现在需要将盒子移上去。

设置wrapper栏的padding值，左右留空100px。

现在需要将left移动到middle左边，right移动到middle右边。

left：设置为相对定位，向左偏移100%（到middle到位置），再偏移100px（到middle之前）。

当 position 设置为 relative 时，right 属性指定了元素的右边界离开其正常位置的偏移。

right:也是同理。

圣杯布局至此完成。

总结：圣杯布局就是基本布局后向左浮动，middle栏留出两遍位置，使用相对定位将左右两栏通过margin-left定位到相对位置。

双飞翼布局：

双飞翼布局和圣杯布局前期基本一致。样式也需要向左浮动。

不同的是，不使用相对定位，而是直接通过margin-left进行调整。

此时，middle的内容被挡住了。

使用圣杯布局，可以设置padding值解决问题。但双飞翼直接设置padding，会导致整个布局乱套。

所以需要再middle栏里再套一层div。

然后再该层加上padding。

双飞翼布局完成。

相同点：

  • 圣杯布局和双飞翼布局都是三列布局
  • 两侧内容宽度固定，中间内容宽度自适应的三栏布局。中间一栏放在文档流前面以优先加载、渲染。
  • 使用float浮动向左脱离文档流，让左中右三列浮动，通过父外边距形成三列布局。

两者区别：

  • 实现方法不同：

    • 圣杯：float+margin+relative相对定位调整位置
    • 双飞翼：float+marign

  • 处理两列位置方法不同：

    • 圣杯：给外部容器加padding，通过相对定位把两边定位出来
    • 双飞翼：在中间这层之外套一层div，加padding讲中间内容挤出。

[css] 在页面上隐藏元素的方法有哪些？

占位:

  • visibility: hidden;
    • visibility：显示或隐藏元素而不改变布局
  • margin-left: -100%;
  • left/right/top/bottom: 9999px/-9999px;
    • position为absolute的情况下，让元素在视觉区外。
  • z-index: -9999
    • 放到最底层，让其他元素把该元素遮掉
  • opacity: 0;
  • transform: scale(0);
    • transform：通过修改css视觉格式化模型的坐标空间，实现旋转、缩放、倾斜、平移。
    • scale() 用于修改元素的大小。

不占位:

  • display: none;
    • 页面不会渲染。可以减少首屏渲染的时间，但是会引起回流和重绘。
      • 重绘：不会引起dom结构和页面布局的变化，只有样式产生改变
      • 回流：引起dom结构和页面布局的变化。有回流一定有重绘。
  • width: 0; height: 0; overflow: hidden;
    • overflow 是 CSS 的简写属性，其设置了元素溢出时所需的行为（即当元素的内容太大而无法适应它的区块格式化上下文时。）

仅对块内文本元素:

  • text-indent: -9999px;
  • font-size: 0;

[html] HTML5的文件离线存储怎么使用，工作原理是什么？

H5的存储方式：

  • Cookies

    • 存在客户端浏览器上
    • 缺点：请求头带数据，大小限制在4K以内（太小）。不安全。每次http请求都会发送到服务端，影响获取资源的效率。需要自己封装获取、 设置、删除cookie的方法。

  • LocalStorage

    • 键值对存储（key-value）。除非手动删除，永不失效

    • 存储容量：每个域名5M

    • 使用方法：

      常用的API

      getItem //取记录

      setItem //设置记录

      removeItem //移除记录

      key //取key所对应的值

      clear //清除记录

  • SessionStorage

    • 会话结束后（关闭页面），数据被清空。
    • 使用方法和LocalStorage相同。

  • 离线缓存 application cache

    • 参考1、参考2

    • 基于一个新建的.appcache文件，通过这个文件上的解析清单离线存储资源，这些资源就会像cookie一样被存储了下来。

    • 优点：无网络可浏览、加快资源的加载速度、减少服务器的负载

    • 关于manifest（即.appcache文件）

      manifest 文件是简单的文本文件，它告知浏览器被缓存的内容（以及不缓存的内容）。
      manifest 文件可分为三个部分：

      • CACHE MANIFEST - 在此标题下列出的文件将在首次下载后进行缓存
      • NETWORK - 在此标题下列出的文件需要与服务器的连接，且不会被缓存
      • FALLBACK - 在此标题下列出的文件规定当页面无法访问时的回退页面（比如 404 页面）
        在线的情况下,用户代理每次访问页面，都会去读一次manifest.如果发现其改变, 则重新加载全部清单中的资源。

      例，一个完整的manifest文件：

    • 和LocalStorage的用法区别：

      Application Cache：通常用于静态资源（静态页面）的缓存。

      LocalStorage：通常用于AJAX请求缓存，存储非关键性AJAX数据。

  • Web SQL

    Web SQL数据库API并不是HTML5规范的一部分，但它是一个独立的规范，引入了一组使用SQL操作客户端数据库的APIs。

  • IndexedDB

    索引数据库（IndexedDB）API（作为HTML5的一部分）对创建具有丰富本地存储数据的数据密集型的离线HTML5 Web应用程序很有用，同时它还有助于本地缓存数据，使传统在线Web应用程序（比如移动Web应用程序）能够快速的运行和响应。

[html] 简述超链接target属性的取值和作用

a标签的target属性，指定在何处显示链接的url，作为浏览上下文的名称。

以下关键词对加载 URL 的位置有特殊含义：

  • _self：当前页面加载。（默认）
  • _blank：通常在新标签页打开，但用户可以通过配置选择在新窗口打开。
  • _parent：当前浏览环境的父级浏览上下文。如果没有父级框架，行为与 _self 相同。
  • _top：最顶级的浏览上下文（当前浏览上下文中最“高”的祖先）。如果没有祖先，行为与 _self 相同。
  • framename：在指定的框架中打开被链接文档
  • ‘任意字符’：若该链接不是已打开的页面，则在新窗口中打开，与_blank一致；若该链接已经打开，则跳转到该标签页并刷新页面。

[html] iframe框架都有哪些优缺点？

优点：

  • 可以实现异步刷新，不影响整体窗口的刷新
  • 可实现跨域，每个iframe框架的源都可以不相同
  • 多页面应用时，对于共同的 header, footer 可以使用 iframe 加载，拆分代码（导航栏的应用）

缺点：

  • 不利于seo
  • 每一个iframe对应一个页面，意味着多余的css、js文件的载入，会增加请求的开销
  • 如果 iframe 内还有滚动条，会严重影响用户体验
  • window.onload 事件会在所有 iframe 加载完成后才触发，因此会造成页面阻塞

算法题

洗完牌抽5张判断是否为同花顺

题目：从扑克牌中随机抽5张牌，判断是不是一个顺子，即这5张牌是不是连续的。2-10为数字本身，A为1，J为11，Q为12，K为13，而大小王可以看成任意数字。

参考思路：

思路一:

我们需要把扑克牌的背景抽象成计算机语言。

我们可以把5张牌看成由5个数字组成的数组。大小王是特殊的数字，我们不妨把它们都当成0，这样和其他扑克牌代表的数字就不重复了。

接下来我们来分析怎样判断5个数字是不是连续的。

最直观的是，我们把数组排序。

由于0可以当成任意数字，我们可以用0去补满数组中的空缺。也就是排序之后的数组不是连续的，即相邻的两个数字相隔若干个数字，但如果我们有足够的0可以补满这两个数字的空缺，这个数组实际上还是连续的。

举个例子，数组排序之后为{0，1，3，4，5}。在1和3之间空缺了一个2，刚好我们有一个0，也就是我们可以它当成2去填补这个空缺。

于是我们需要做三件事情：把数组排序，统计数组中0的个数，统计排序之后的数组相邻数字之间的空缺总数。

如果空缺的总数小于或者等于0的个数，那么这个数组就是连续的；反之则不连续。最后，我们还需要注意的是，如果数组中的非0数字重复出现，则该数组不是连续的。换成扑克牌的描述方式，就是如果一副牌里含有对子，则不可能是顺子。

思路二:

1）确认5张牌中除了0，其余数字没有重复的（可以用表统计的方法）;

\2) 满足这样的逻辑：（max，min分别代表5张牌中的除0以外的最大值最小值）

如果没有0，则max-min=4，则为顺子，否则不是

如果有一个0，则max-min=4或者3，则为顺子，否则不是

如果有两个0，则max-min=4或者3或者2，则为顺子，否则不是最大值和最小值在1）中就可以获得，这样就 不用排序了
