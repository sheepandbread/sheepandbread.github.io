about git

命令

git commit：提交

git branch xxx：新建分支

git checkout xxx：切换到xxx分支（更改head指向）

git checkout -f xxx xxx2：强制修改分支

git merge xxx：合并分支，存在两个

git rebase xxx：合并分支，存在一个

git rebase -i xxx：合并，修改记录排序

git log：显示日志

撤销变更

  • git reset
  • git revert

相对引用：

  • ^向上移动1步
  • ~n向上移动n步

git cherry-pick <提交号>：将选定的分支提交合并到当前分支中。

git tag tagname XX：建立标签，指向提交记录XX

git bisect：

git describe <ref>：描述离你最近的锚点

  • <ref> 可以是任何能被 Git 识别成提交记录的引用，如果你没有指定的话，Git 会使用你目前所在的位置（HEAD）

  • 它输出的结果：

  • tag 表示的是离 ref 最近的标签， numCommits 是表示这个 ref 与 tag 相差有多少个提交记录， hash 表示的是你所给定的 ref 所表示的提交记录哈希值的前几位。

git clone：克隆远程仓库

git fetch：从远程仓库获取数据

  • git fetch origin <source>:<destination>

git pull：git fetch和git merge的合体

  • git pull --rebase 就是 fetch 和 rebase 的简写！

  • git pull origin foo 相当于：git fetch origin foo; git merge o/foo

  • git pull origin bar~1:bugFix 相当于：git fetch origin bar~1:bugFix; git merge bugFix

git push：数据上交合并到远程仓库

  • git push <remote> <place>

    • git push origin main

      切到本地仓库中的“main”分支，获取所有的提交，再到远程仓库“origin”中找到“main”分支，将远程仓库中没有的提交记录都添加上去

  • git push origin <source>:<destination>

    • 这个参数实际的值是个 refspec，“refspec” 是一个自造的词，意思是 Git 能识别的位置（比如分支 foo 或者 HEAD~1）

git checkout -b A o/main：创建一个名为 A 的分支，它跟踪远程分支 o/main

git branch -u o/main foo：foo分支追踪远程分支o/main

特殊用法

  • git push origin :<destination>：删除远程仓库的destination分支
  • git fetch origin :<destination>：本地创建新分支destination

上传项目到github

一个新项目

情况一：本地创建新存储库并推到github上：

  • git init：初始化一个全新的git存储库
  • git add：暂存更改。告诉 Git 哪些文件的修改应该包含在下一次提交（commit）中。
  • git commit：提交暂存区里面的内容。
  • git remote add：新增远程。
    • 两个参数：远程名称（例如origin），远程url（项目地址）
  • git push：将本地分支上的提交推送到远程存储库。
    • 该命令有两个参数：
      • origin——远程名称
      • main——分支名称
