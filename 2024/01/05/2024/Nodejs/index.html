Nodejs

简介&下载

Node.js是一个开源的、跨平台的JavaScript运行环境

作用：

  1. 开发服务器应用
  2. 开发工具类应用：webpack、vite、Babel
  3. 开发桌面端应用：VSCode、Figma、Postman

下载：

  • 官网：https://nodejs.org/en
  • LTS为长期维护版本

Node.js的注意点：

  • DOM、BOM等api无法使用。
  • console和定时器可使用。
  • 顶级对象为global（可使用globalThis访问顶级对象）。

Buffer

Buffer：缓冲区。是一个类似于数组的对象，用于表示固定长度的字节序列。（一段内存空间，专门处理二进制数据）

  • 特点
    • Buffer大小 固定且无法调整
    • Buffer性能较好，可以直接对计算机内存进行操作
    • 每个元素的大小为1字节（byte）
  • 创建（三个方法）
    1. Buffer.alloc() 方法创建指定大小的新缓冲区对象。
      • 创建的二进制会归零
    2. Buffer.allocUnsafe() 方法创建指定大小的新缓冲区对象。
      • 与 Buffer.alloc() 方法不同，因为它创建了一个未预填充的缓冲区，并且可能包含来自旧缓冲区的信息。这就是为什么它 不安全。
      • 但相比alloc，它会更快
    3. Buffer.from()方法用于创建包含指定字符串，数组或缓冲区的新缓冲区。
      • 将字符串或数组转换成二进制存储
  • buffer和字符串转换：buf.toString()
    • 转换后的字符串为UTF-8格式
    • 中文的utf-8格式占三个字节

fs模块

fs模块：file system（文件系统）。实现与硬盘的交互，例如文件的创建、删除、重命名、移动。还有文件内容的写入、读取，以及文件夹的相关操作。

__dirname：表示所在文件的所在目录的绝对路径。

__filename：表示文件的绝对路径。

一、文件写入

需要持久化文件保存数据的时候，可以用到文件写入。

应用场景：下载文件、安装软件、保存程序日志（如git）、编辑器保存文件等。

  • 写入文件（异步）：fs.writeFile(file, data[, options], callback)

    若写入文件不存在，会创建。

    • file - 文件名或文件描述符。
    • data - 要写入文件的数据，可以是 String(字符串) 或 Buffer(缓冲) 对象。
    • options - 该参数是一个对象，包含 {encoding, mode, flag}。默认编码为 utf8, 模式为 0666 ， flag 为 ‘w’
    • callback - 回调函数，回调函数只包含错误信息参数(err)，在写入失败时返回.
    • 同步版：fs.writeFileSync(filename, data[, options])

  • 追加写入：

    • 异步版：fs.appendFile(filename, data[, options], callback)
    • 同步版：fs.appendFileSync(filename, data[, options])
    • writeFile也可以实现追加写入，flag设置为a即可。

  • 流式写入：适合大文件写入或频繁写入

    • 创建writestream对象： fs.createWriteStream(path[, options])
    • 写入：ws.write()`
    • 关闭文档：ws.close()

案例：

  • 打开文件（异步）:fs.open(path, flags[, mode], callback)

    • flags - 文件打开的行为（例如r为读取模式，w为写入模式，a为追加模式等）
    • mode - 设置文件权限（默认为0666【可读，可写】）

三、文件移动和重命名

  • 重命名和文件移动：fs.rename(oldPath, newPath, callback)

本质就是修改路径。

四、文件删除

  • 文件删除unlink
    • 异步：fs.unlink(path, callback)
    • 同步：fs.unlinkSync(path)
  • 文件/目录递归删除rm
    • 异步：fs.rm( path, options, callback );
    • 同步：fs.rmSync( path, options );

五、文件夹操作

  • 创建文件夹
    • 异步：fs.mkdir(path[, options], callback)
      • path - 文件路径。
      • options 参数可以是：
        • recursive - 是否以递归的方式创建目录，默认为 false。
        • mode - 设置目录权限，默认为 0777。
      • callback - 回调函数，没有参数。
  • 读取文件夹
    • 异步：fs.readdir(path, callback)
    • 同步：fs.readdirSync(path)
  • 删除文件夹
    • 异步：fs.rmdir(path, callback)
      • 可以递归删除，设置{recursive:true}。但不建议。
    • 同步：fs.rmdirSync(path)

六、查看资源信息

  • 获取文件信息（异步）：fs.stat(path, callback)
    • path - 文件路径。
    • callback - 回调函数，带有两个参数如：(err, stats), stats 是 fs.Stats 对象。

HTTP协议

协议：双方必须遵从的一种规定。

HTTP：Hypertext Transfer Protocol（超文本传输协议）。

请求报文分析：

  • 请求行： 方法 URL HTTP版本号
    • 常见方法：GET、POST、PUT/PATCH（更新数据）、DELETE……
    • URL：Uniform Resource Locator（统一资源定位符）
      • 用于定位服务器的资源。
      • 组成：协议名+主机名+[端口号]+路径+[查询字符串]
  • 请求头：由一系列键值对组成。记录浏览器的相关信息。
  • 请求体

响应报文和请求报文格式类似，由响应行、响应头、响应体组成：

  • 响应行：HTTP/1.1 200 OK

    • HTTP/1.1：HTTP协议版本号
    • 200：响应状态码
    • 404 Not Found 500 Internal Server Error
      • 还有一些状态码，参考： https://developer.mozilla.org/zh-CN/docs/Web/HTTP/Status
    • OK：响应状态描述

  • 响应头

IP地址：一个数字标识。用来表示网络中的设备，实现设备间通信。

  • 分类
    • 本机回环IP：127.0.0.1 - 127.255.255.254
    • 局域网（私网）IP
      • 10.0.0.0 - 10.255.255.255
      • 172.16.0.0 - 172.31.255.255
      • 192.168.0.0 - 192.168.255.255（用的较多）
    • 广域网（公网）IP

端口：应用程序的数字标识。一台现代计算机有65535个端口。实现不同主机应用程序之间的通信。
